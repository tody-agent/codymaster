## 🛑 아무도 이야기하지 않는 문제

귀하는 AI 코딩 에이전트를 설치했습니다. 아주 *훌륭합니다*. 어떤 인간보다 코드를 빠르게 작성하죠.

하지만 곧 현실적인 문제에 직면하게 됩니다:

| 😤 실제로 일어나는 일 | 💀 실제 비용 |
|--------------------------|-----------------|
| AI는 **매번 다르게 디자인합니다** — 같은 브랜드임에도 3가지 다른 스타일 | 고객은 귀하를 3개의 다른 회사라고 생각합니다 |
| AI가 버그 하나를 고치면, **조용히 다른 5가지 기능을 망가뜨립니다** | 같은 작업을 3~4번 반복하게 됩니다 |
| AI는 세션 사이에 **모든 것을 잊어버립니다** | 매일 아침 같은 코드베이스를 다시 설명해야 합니다 |
| AI는 테스트도 문서도 작성하지 않습니다 | 귀하의 코드베이스는 사상누각이 됩니다 |
| 15개의 서로 다른 스킬을 설치하지만 — **그 중 어느 것도 서로 연동되지 않습니다** | 시너지가 전혀 없는 프랑켄슈타인 툴킷 |
| 운영 환경 배포 = **배포하고 기도하기** 🙏 | 새벽 2시에 배포가 중단되고, 롤백도 안 됩니다 |

> *"AI는 저에게 100개의 손을 주었습니다. 하지만 규율이 없으니 그 손들은 혼란을 만들었습니다."*
> — **Tody Le**, Head of Product · 10년 이상의 경력 · CodyMaster 제작자

## 🟢 해결책: 하나의 키트에 담긴 시니어 팀 전체

CodyMaster는 단순한 "또 다른 AI 스킬 팩"이 아닙니다. 10년 이상의 제품 관리 경험과 6개월간 실전에서 검증된 바이브 코딩(vibe coding)을 68+개의 상호 연결된 스킬로 응축하여 **하나의 통합된 시스템**으로 작동하게 만든 것입니다.

CodyMaster를 설치하는 것은 단순히 스킬을 추가하는 것이 아닙니다.

### 🧠 통합 두뇌: 5계층 메모리 아키텍처 + Smart Spine

귀하의 AI는 단지 실행만 하는 것이 아닙니다 — 세션과 기기 간에 지속되는 5계층 + Smart Spine 시스템을 사용하여 **이해하고 기억합니다**:

1. **Sensory Memory (세션)** — 활성 파일 및 터미널의 즉각적인 컨텍스트.
2. **Working Memory (`cm-continuity`)** — 세션 간 스크래치 패드. AI는 동일한 실수를 절대 반복하지 않습니다.
3. **Long-Term Memory (`learnings.json`)** — 스마트 에빙하우스 TTL 감쇠가 적용된 강화된 학습 내용.
4. **Semantic Memory (`cm-deep-search`)** — `qmd`를 사용한 문서 간 로컬 벡터 검색.
5. **Structural Memory (`cm-codeintell`)** — AST 기반 코드 그래프(CodeGraph). 전체 코드베이스 컨텍스트에 대해 최대 95%의 토큰 압축률.

🦴 **Smart Spine (v4.5+)** — 모든 5계층을 연결하는 신경 시스템:
- **SQLite + FTS5** — BM25 랭킹 키워드 검색으로 플랫 JSON 스캨 대체.
- **Progressive Loading (L0/L1/L2)** — 컨텍스트가 최소 비용으로 로드됨. 78% 토큰 절감.
- **cm:// URI Scheme** — 스킬이 파일 경로 대신 URI로 컨텍스트 요청.
- **Token Budget** — 200k 토큰 창이 카테고리별로 사전 할당. 더 이상 조용한 오버플로우 없음.
- **Context Bus** — 스킬이 체인에서 실시간으로 출력 공유.
- **MCP Server** — Claude Desktop 및 모든 MCP 클라이언트를 지원하는 7개 도구.

☁️ **The Cloud Brain (`cm-notebooklm`)**
가치가 높은 지식과 디자인 패턴은 NotebookLM에 동기화되어 프로젝트에 기기 간에 공유되는 보편적인 "영혼"을 제공합니다. 팟캠스트와 플래시카드를 자동 생성하여 AI와 함께 일하는 인간 개발자의 온보딩을 돕습니다.

📖 [전체 지식 아키텍처 읽기 (EN) →](docs/architecture/knowledge-architecture.md)

### 🎨 디자인 시스템 추출 — 오래된 제품에서도 가능합니다

디자인 시스템이 없는 레거시 제품이 있나요? **cm-design-system**이 웹사이트를 스캔하여 색상, 타이포그래피, 간격 및 토큰을 추출한 다음, 적절한 디자인 시스템을 다시 구축합니다. 단 한 줄의 코드를 작성하기 전에 **Pencil.dev** 또는 **Google Stitch**를 통해 디자인을 시각적으로 미리 확인하세요.

### 📝 문서가 하나도 없나요? 문제없습니다.

기존 코드가 무엇을 하는지 모르시나요? **`cm-dockit`**이 전체 코드베이스를 읽고 다음을 생성합니다:
- 📚 기술 아키텍처 문서
- 📖 사용자 가이드 및 SOP
- 🔌 API 참조서
- 🎯 페르소나 분석 및 JTBD 매핑
- 🌐 다국어 지원. SEO 최적화.

**한 번의 스캔 = 완벽한 지식 베이스.**

### 💡 전략적 브레인스토밍 (Design Thinking + 9 Windows)

복잡한 요청에 대한 코딩을 시작하기 전에 **`cm-brainstorm-idea`**가 다차원 분석(기술, 제품, 디자인, 비즈니스)을 통해 제품을 평가합니다. 9 Windows (TRIZ) 프레임워크를 사용하여 2-3개의 검증된 옵션을 생성하고, 세부 계획 전에 방향을 확인하기 위해 **Pencil.dev** 또는 **Google Stitch**를 통한 시각적 UI 미리보기를 제공합니다.

📖 [UI 미리보기 단계에 대해 더 알아보기 →](docs/workflows/brainstorm-ui-preview.md)

### 🏭 AI Content Factory v2.0 & 비주얼 대시보드

콘텐츠 확장이 필요하신가요? **`cm-content-factory`**는 자가 학습이 가능한 다중 에이전트 콘텐츠 엔진입니다. 콘텐츠 마스터리 프레임워크 (StoryBrand + Cialdini)를 통해 자동으로 조사, 작성, 감사(SEO 및 설득력 검증)를 수행하고 전환율이 높은 기사를 배포하여 결과를 보장합니다.

**비주얼 대시보드** (`cm-dashboard`)에서 모든 것을 추적하세요: 더 이상 추측할 필요가 없습니다. 실시간 칸반 보드에서 모든 작업, 모든 에이전트, 모든 배포를 추적하세요. 파이프라인 진행 상황, 토큰 추적기, 이벤트 로그를 모두 한 화면에서 볼 수 있습니다.

## 🆚 흩어진 스킬들 vs CodyMaster

| | 😵 15개의 무작위 스킬들 | 🧠 CodyMaster |
|---|---|---|
| **통합** | 각 스킬이 독립적이며 컨텍스트 공유 없음 | 체인으로 연결되고 메모리를 공유하며 소통하는 68+개의 스킬 |
| **라이프사이클** | 코딩만 지원 | 아이디어 → 디자인 → 코드 → 테스트 → 배포 → 문서화 → 학습까지 지원 |
| **메모리** | 세션 사이에 모든 것을 잊음 | 5계층 통합 두뇌 시스템: Sensory → Working → Long-term → Semantic → Structural + Cloud Brain |
| **안전성** | YOLO 배포 | 4레이어 보호: TDD → 보안 → 격리 → 다중 게이트 배포 |
| **디자인** | 매번 무작위 UI | 디자인 시스템 추출 및 강제 적용 + 시각적 미리보기 |
| **문서화** | "나중에 README나 써야지" | 코드에서 완전한 문서, SOP, API 참조를 자동 생성 |
| **자기 개선** | 정적 — 설치한 것만 사용 가능 | 실수로부터 배우고, 자동으로 새로운 스킬을 발견하며 매일 더 똑똑해짐 |
| **유지보수** | 15개의 저장소를 별도로 업데이트 | 단 한 번의 `git pull`로 모든 것을 업데이트 |

## 🦥 게으른 사람들을 위해 만들어졌습니다 (진심으로요)

솔직히 말씀드리면: **CodyMaster는 게으른 사람들을 위해 만들어졌습니다.**

다음을 원하신다면:
- ✅ 채팅 메시지를 입력하고 **작동하는 제품**을 결과로 받기
- ✅ AI가 **실수로부터 배우고** 매일 더 발전하기
- ✅ 동일한 보일러플레이트를 두 번 다시 설정하지 않기
- ✅ 기도하는 대신 **확신**을 가지고 배포하기

**→ CodyMaster가 정답입니다.**

다음을 선호하신다면:
- ❌ AI 출력의 모든 라인을 수동으로 검토하기
- ❌ 모든 프로젝트마다 동일한 설정 의식을 반복하기
- ❌ 안전장치 없는 느리고 수동적인 배포

**→ CodyMaster는 당신을 위한 것이 아닙니다.**

## 🎮 명령어

**CLI 명령어:**
```
cm                          → Cody 🐹와 함께하는 빠른 메뉴
cm task add "..."           → 작업 추가
cm task list                → 작업 보기
cm status                   → 프로젝트 건강 상태
cm dashboard                → 미션 컨트롤 열기
cm continuity index         → L0 메모리 인덱스 재생성
cm continuity budget        → 토큰 예산 할당 보기
cm continuity bus           → context bus 상태 보기
cm continuity mcp           → MCP 서버 구성 출력
cm continuity migrate       → JSON → SQLite 마이그레이션
cm continuity export        → SQLite → JSON 내보내기
cm resolve <uri>            → 모든 cm:// URI 해석
```

