## 🛑 아무도 이야기하지 않는 문제

귀하는 AI 코딩 에이전트를 설치했습니다. 아주 *훌륭합니다*. 어떤 인간보다 코드를 빠르게 작성하죠.

하지만 곧 현실적인 문제에 직면하게 됩니다:

| 😤 실제로 일어나는 일 | 💀 실제 비용 |
|--------------------------|-----------------|
| AI는 **매번 다르게 디자인합니다** — 같은 브랜드임에도 3가지 다른 스타일 | 고객은 귀하를 3개의 다른 회사라고 생각합니다 |
| AI가 버그 하나를 고치면, **조용히 다른 5가지 기능을 망가뜨립니다** | 같은 작업을 3~4번 반복하게 됩니다 |
| AI는 세션 사이에 **모든 것을 잊어버립니다** | 매일 아침 같은 코드베이스를 다시 설명해야 합니다 |
| AI는 테스트도 문서도 작성하지 않습니다 | 귀하의 코드베이스는 사상누각이 됩니다 |
| 15개의 서로 다른 스킬을 설치하지만 — **그 중 어느 것도 서로 연동되지 않습니다** | 시너지가 전혀 없는 프랑켄슈타인 툴킷 |
| 운영 환경 배포 = **배포하고 기도하기** 🙏 | 새벽 2시에 배포가 중단되고, 롤백도 안 됩니다 |

> *"AI는 저에게 100개의 손을 주었습니다. 하지만 규율이 없으니 그 손들은 혼란을 만들었습니다."*
> — **Tody Le**, Head of Product · 10년 이상의 경력 · CodyMaster 제작자

## 🟢 해결책: 하나의 키트에 담긴 시니어 팀 전체

CodyMaster는 단순한 "또 다른 AI 스킬 팩"이 아닙니다. 10년 이상의 제품 관리 경험과 6개월간 실전에서 검증된 바이브 코딩(vibe coding)을 33개의 상호 연결된 스킬로 응축하여 **하나의 통합된 시스템**으로 작동하게 만든 것입니다.

CodyMaster를 설치하는 것은 단순히 스킬을 추가하는 것이 아닙니다.
**시니어 팀 전체를 고용하는 것입니다:**

## ⚡ CodyMaster가 다른 점

다른 스킬 팩들은 파편화된 도구들을 제공합니다. CodyMaster는 귀하의 AI를 위한 **상호 연결된 운영 체제**를 제공합니다.

### 🔄 전체 라이프사이클 커버리지 (아이디어 → 운영)

공백도, 수동 전달도 없습니다. 모든 단계가 포함됩니다:

```mermaid
graph LR
    A["💡 아이디어"] --> B["📋 계획"]
    B --> C["🎨 디자인"]
    C --> D["🧪 테스트 우선"]
    D --> E["💻 코드"]
    E --> F["🔍 디버그"]
    F --> G["✅ 품질 게이트"]
    G --> H["🔒 보안 스캔"]
    H --> I["🚀 배포"]
    I --> J["📊 모니터링"]
    J --> K["📝 문서화"]
    K --> L["🧠 학습 및 개선"]
    L -.-> A
    classDef phase fill:#353b48,stroke:#fbc531,stroke-width:2px,color:#fff;
    class A,B,C,D,E,F,G,H,I,J,K,L phase;
```

### 🧠 실수로부터 배우는 두뇌

귀하의 AI는 단지 실행만 하는 것이 아닙니다 — **기억하고 개선합니다**:

- **`cm-continuity`** — 세션 간 작업 기억. AI는 무엇이 잘못되었는지 기억하고 동일한 실수를 절대 반복하지 않습니다.
- **`cm-skill-mastery`** — 어떻게 하는지 모르시나요? **자동으로 적절한 스킬을 찾아** 스스로를 업그레이드합니다.
- **`cm-deep-search`** — 200개가 넘는 파일의 코드베이스에서 길을 잃으셨나요? 몇 초 만에 모든 것을 대상으로 시맨틱 검색을 수행합니다.

### 🛡️ 다중 레이어 보호 (코드베이스가 파괴되지 않도록 보호합니다)

모든 코드 라인은 프로덕션에 도달하기 전에 여러 안전 게이트를 통과합니다:

```mermaid
flowchart LR
    subgraph "Layer 1: Write"
        A["cm-tdd<br/>Tests First"] --> B["cm-code-review"]
    end
    subgraph "Layer 2: Secure"
        B --> C["cm-secret-shield<br/>Leak Scan"] --> D["cm-identity-guard<br/>Right Account"]
    end
    subgraph "Layer 3: Isolate"
        D --> E["cm-git-worktrees<br/>Isolated Branch"]
    end
    subgraph "Layer 4: Deploy"
        E --> F["cm-quality-gate<br/>Evidence Check"] --> G["cm-safe-deploy<br/>Multi-Gate Pipeline"]
    end
    style A fill:#e84118,stroke:#c23616,color:#fff
    style C fill:#e84118,stroke:#c23616,color:#fff
    style F fill:#0097e6,stroke:#00a8ff,color:#fff
    style G fill:#4cd137,stroke:#44bd32,color:#fff
```

> **결과:** 비밀 유출 제로. 잘못된 계정 배포 제로. "내 컴퓨터에서는 됐는데" 식의 실패 제로.

### 🎨 디자인 시스템 추출 — 오래된 제품에서도 가능합니다

디자인 시스템이 없는 레거시 제품이 있나요? **cm-design-system**이 웹사이트를 스캔하여 색상, 타이포그래피, 간격 및 토큰을 추출한 다음, 적절한 디자인 시스템을 다시 구축합니다. 단 한 줄의 코드를 작성하기 전에 **Pencil.dev** 또는 **Google Stitch**를 통해 디자인을 시각적으로 미리 확인하세요.

### 📝 문서가 하나도 없나요? 문제없습니다.

기존 코드가 무엇을 하는지 모르시나요? **`cm-dockit`**이 전체 코드베이스를 읽고 다음을 생성합니다:
- 📚 기술 아키텍처 문서
- 📖 사용자 가이드 및 SOP
- 🔌 API 참조서
- 🎯 페르소나 분석 및 JTBD 매핑
- 🌐 다국어 지원. SEO 최적화.

**한 번의 스캔 = 완벽한 지식 베이스.**

### 📊 비주얼 대시보드 — 한눈에 모든 것을 확인하세요

더 이상 추측할 필요가 없습니다. 실시간 칸반 보드에서 모든 작업, 모든 에이전트, 모든 배포를 추적하세요. 파이프라인 진행 상황, 토큰 추적기, 이벤트 로그를 모두 한 화면에서 볼 수 있습니다.

## 🆚 흩어진 스킬들 vs CodyMaster

| | 😵 15개의 무작위 스킬들 | 🧠 CodyMaster |
|---|---|---|
| **통합** | 각 스킬이 독립적이며 컨텍스트 공유 없음 | 체인으로 연결되고 메모리를 공유하며 소통하는 34개의 스킬 |
| **라이프사이클** | 코딩만 지원 | 아이디어 → 디자인 → 코드 → 테스트 → 배포 → 문서화 → 학습까지 지원 |
| **메모리** | 세션 사이에 모든 것을 잊음 | 4단계 메모리 시스템: Working → Episodic → Semantic → Deep Search |
| **안전성** | YOLO 배포 | 4레이어 보호: TDD → 보안 → 격리 → 다중 게이트 배포 |
| **디자인** | 매번 무작위 UI | 디자인 시스템 추출 및 강제 적용 + 시각적 미리보기 |
| **문서화** | "나중에 README나 써야지" | 코드에서 완전한 문서, SOP, API 참조를 자동 생성 |
| **자기 개선** | 정적 — 설치한 것만 사용 가능 | 실수로부터 배우고, 자동으로 새로운 스킬을 발견하며 매일 더 똑똑해짐 |
| **유지보수** | 15개의 저장소를 별도로 업데이트 | 단 한 번의 `git pull`로 모든 것을 업데이트 |

## 🦥 게으른 사람들을 위해 만들어졌습니다 (진심으로요)

솔직히 말씀드리면: **CodyMaster는 게으른 사람들을 위해 만들어졌습니다.**

다음을 원하신다면:
- ✅ 채팅 메시지를 입력하고 **작동하는 제품**을 결과로 받기
- ✅ AI가 **실수로부터 배우고** 매일 더 발전하기
- ✅ 동일한 보일러플레이트를 두 번 다시 설정하지 않기
- ✅ 기도하는 대신 **확신**을 가지고 배포하기

**→ CodyMaster가 정답입니다.**

다음을 선호하신다면:
- ❌ AI 출력의 모든 라인을 수동으로 검토하기
- ❌ 모든 프로젝트마다 동일한 설정 의식을 반복하기
- ❌ 안전장치 없는 느리고 수동적인 배포

**→ CodyMaster는 당신을 위한 것이 아닙니다.**
