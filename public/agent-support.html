CodyMaster Claw lets you write one AGENTS.md manifest and auto-generate configs for OpenClaw, OpenFang, Claude Desktop, Cursor, Manus, MaxClaw, and Gemini Antigravity. No more per-tool setup. One source of truth for all your AI coding agents.

Get Started in 3 Commands

From zero to fully-configured AI workspace in under 60 seconds. Works with any project, any stack.

Step 1: Install CodyMaster

Terminal
npm install -g cody-master

Step 2: Bootstrap Your Project

Terminal
cody bootstrap

Step 3: Start Any Agent

Terminal
openclaw . # or cursor . / claude .

That's it. AGENTS.md + per-tool configs are auto-generated. Every agent reads the same rules.

🏢 Enterprise — Governance & Compliance

Define security policies once in AGENTS.md: "Never commit secrets", "Always run OWASP scan", "Enforce i18n for all user-facing strings". Every AI agent in your org automatically inherits these rules.

🔄 Migration — Switch Agents Without Pain

Testing a new AI tool? Just run cody bootstrap again. It regenerates configs for the new tool while keeping your AGENTS.md intact. Zero migration overhead. Compare tools with the same codebase rules.

The AGENTS.md Open Standard

Based on the Microsoft Build 2025 specification, AGENTS.md is the single file that tells every AI agent how your project works — its tech stack, rules, commands, and identity.

  • Identity & Context: Tech stack, deploy targets, brand guidelines — loaded instantly by any agent.
  • Universal Commands: Standardizes dev, build, test, and deploy across all tools.
  • Guardrails: "Never deploy without gates", "Always translate strings", "TDD is mandatory".
  • Per-Agent Mapping: Auto-generates .clawrc, CLAUDE.md, .cursorrules, .gemini/settings.json from one manifest.
AGENTS.md
# Project Manifest

## Identity
- Name: my-saas-app
- Stack: Next.js + Supabase
- Deploy: Cloudflare Pages

## Guardrails
1. TDD mandatory (cm-tdd)
2. 6-gate check before deploy
3. Never hardcode i18n strings
4. Identity verification on push

## Commands
| Action | Command |
|--------|---------|
| Dev | npm run dev |
| Test | npm run test |
| Gate | npm run test:gate |
| Deploy | npm run deploy |

Best Practices

Follow these patterns to get the most out of CodyMaster Claw.

Version Your AGENTS.md

Check it into Git. Every team member and every AI agent reads the same version. Treat it like your .eslintrc — it defines the law of the land.

Start with Guardrails

Define rules before features. "TDD mandatory", "No deploy without gates", "i18n all strings". These constraints paradoxically make AI faster by eliminating rework.

Use Per-Agent Overrides

Each tool has its own strengths. Use AGENTS.md for shared rules, but let per-tool configs (CLAUDE.md, .cursorrules) handle agent-specific optimizations.

Run Bootstrap on CI

Add cody bootstrap --check to your CI pipeline to verify AGENTS.md stays in sync with generated configs. Catch drift before it causes confusion.

Keep Identity Strict

Define Git account, Cloudflare account, and Supabase project in AGENTS.md. CodyMaster's cm-identity-guard blocks wrong-account deploys before they happen.

Document Custom Skills

List your custom skills in AGENTS.md so any agent knows what specialized behaviors are available. Skills are portable across all supported platforms.
