## Dependency Order

```
Phase 1:
  1.1 (L0 Indexer) ──→ can start immediately
  1.2 (Context Bus) ──→ can start immediately (parallel with 1.1)
  1.3 (Token Budget) ──→ depends on 1.1 (needs L0 indexes to switch to)
  1.4 (Integration) ──→ depends on 1.1 + 1.2 + 1.3

Phase 2:
  2.1 (SQLite) ──→ can start after Phase 1 complete
  2.2 (Migration) ──→ depends on 2.1
  2.3 (URI Resolver) ──→ depends on 2.1
  2.4 (MCP Server) ──→ depends on 2.1 + 2.3
  2.5 (File Watcher) ──→ depends on 2.1
  2.6 (Integration) ──→ depends on 2.1-2.5
```

## Execution Strategy

**Phase 1:** Sequential — build L0 indexer first (foundation), then context bus (parallel), then token budget (needs L0).

**Phase 2:** Parallel where possible — SQLite layer first, then URI resolver + migration + file watcher can proceed in parallel. MCP server last (depends on all).

**Suggested cm-execution mode:** Mode A (Sequential) for Phase 1, Mode C (Parallel) for Phase 2 sub-tasks.
