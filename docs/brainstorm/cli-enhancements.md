## Qualified Problem
**For:** Devs & AI Agents using CodyMaster
**Who:** currently use the CLI for basic project/task/deploy management
**The:** `cm` CLI is the main gateway to the platform
**That:** needs to expose the new, powerful intelligence layers (Memory, Deep Search, Brain Indexing, Skill Creation, and Security Gates) natively to the terminal
**Unlike:** standard task runners, this CLI dictates AI Agent behavior
**Our approach:** Expand the CLI commands to match the cognitive and operational evolution of the CodyMaster system.

### Root Cause(s):
1. **Feature Lag**: The CLI still focuses heavily on standard Kanban (projects/tasks/deploys) while the system has evolved to include NotebookLM memory, skill chains, and deep indexing.
2. **Hidden Capabilities**: Skills like `skill-creator-ultra` or `cm-secret-shield` are invoked via chat rather than having native CLI triggers.

## Options Evaluated

### Option A: The "Intelligence Operations" Focus (Memory & Skills)
**Approach:** Add commands to manage the "Brain" of CodyMaster. Focus on long-term memory and skill creation.
**What changes:**
- `cm memory <cmd>`: `add`, `list`, `search`, `sync` (triggers NotebookLM compile).
- `cm skill <cmd>`: `create` (triggers skill-creator-ultra), `edit`, `search` (local BM25).
- `cm index`: Triggers `skeleton.md` and `brain.json` generation.

### Option B: The "DevOps & Audit" Focus (Gates & Security)
**Approach:** Add commands to enforce CodyMaster's strict deployment and quality guidelines.
**What changes:**
- `cm audit`: Runs `cm-quality-gate`, `cm-secret-shield`, and frontend safety checks.
- `cm gate <id>`: Manually run specific test pipelines.
- `cm config`: Configure API keys, Webhooks, and Cloudflare/Supabase identities.

### Option C: The "Holistic AI Shell" (Comprehensive Expansion) 🌟
**Approach:** Combine Option A and Option B into a unified, modular CLI expansion.
**What changes:**
1. **Knowledge & Memory**: `cm memory [add/list/sync]`, `cm index [skeleton/brain]`
2. **Skill Ecosystem**: `cm skill [create/install/ls]`
3. **Safety & Ops**: `cm audit [secrets/vulnerabilities]`, `cm test`
4. **Agent Orchestration**: `cm agent [list/status/route]`
**Effort:** Medium
**Risk level:** Low (Easy to add to Commander.js incrementally).

## 🎯 Recommendation
**Choose Option C: The "Holistic AI Shell"**

### Why Option C:
1. It bridges the gap between the robust AI skills you've built (like `skill-creator-ultra`, `cm-secret-shield`) and the developer's terminal experience.
2. Commands like `cm memory sync` or `cm index brain` make the newly developed Semantic Indexing and NotebookLM features fundamentally easier to interact with outside of the chat context.
3. Adding explicit commands standardizes agent behavior (e.g., an agent can run `cm audit` automatically before claiming a task is done).
